# Repository Guidelines

## Структура проекта
- `index.html` — базовая разметка приложения и экраны меню/игры.
- `style.css` — тема и раскладки интерфейса (4‑пробельные отступы).
- `game.js` — основная логика раундов, карт, энтропии и UI-обработчики.
- `dist/` — собранная версия для быстрого запуска из браузера.
- `test.js` — Playwright-смоук: загрузка `index.html`, проверка UI и консольных ошибок.

## Сборка, запуск и разработка
- `npm install` — установка зависимостей (локальный `serve` и Playwright).
- `npm start` — статический сервер на текущем каталоге; откройте указанный порт (обычно 3000).
- `node test.js` — прогон smоke-теста в headless Chromium по локальному файлу `index.html`.
- Билд не требуется (`npm run build` только выводит напоминание).

## Стиль кода и оформление
- JavaScript: 4 пробела, `const`/`let` вместо `var`, краткие чистые функции; избегайте глобальных побочных эффектов вне `DOMContentLoaded`.
- Имена: элементы DOM по ID в `kebab-case`, переменные и функции в `camelCase`, константы можно в `UPPER_SNAKE` при необходимости.
- Комментарии — только по делу (нетривиальная логика, расчёты, побочные эффекты).

## Тестирование
- Основной сценарий — `node test.js`; убедитесь, что нет консольных ошибок и элементы UI отображаются.
- Дополняя тест, придерживайтесь Playwright API, добавляйте явные ожидания (`waitForTimeout` минимизируйте).
- Новые проверки — рядом с существующими секциями в `test.js`, именуйте лог-вывод понятно.

## Коммиты и pull request
- Сообщения коммитов в повелительном наклонении, кратко: `add entropy pulse warning`, `fix mana display`.
- Для PR: краткое описание задачи, список ключевых изменений, как проверить (команда/шаги), приложите скриншоты UI при визуальных правках.
- Держите изменения фокусными; правки фронта сопровождайте быстрым скрином и ссылкой на затронутые элементы (`#menu-screen`, `#game-screen` и т.п.).
